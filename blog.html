Core Architecture

The foundation of Shikimori-MD is built on a modular, event-driven architecture designed for scalability and maintainability.

System Components

Node.js TypeScript WhatsApp Web Protocol MongoDB
  • Event Bus: Centralized event handling system for all WhatsApp events (messages, joins, leaves, etc.)
  • Command Handler: Dynamic loading and execution of commands with permission validation
  • Middleware Pipeline: Pre-processing chain for authentication, rate limiting, and context preparation
  • Database Abstraction: Unified interface for both MongoDB and in-memory caching layers
// Example command structure interface Command { name: string; description: string; category: string; aliases?: string[]; usage?: string; cooldown?: number; permissions?: PermissionLevel[]; execute: (ctx: Context) => Promise<void>; }

Command System

Advanced command processing with support for complex interactions and rich responses.

Key Features

Dynamic Loading Alias Support Cooldowns Permission Levels
  • Modular Commands: Each command is self-contained with its own configuration and logic
  • Context Object: Rich execution context with message metadata, user info, and helper methods
  • Interactive Menus: Button-based navigation for complex command flows
  • Multi-language Support: Built-in i18n system for localized responses
  • Error Handling: Graceful degradation with user-friendly error messages
// Example command execution flow bot.on('message', async (message) => { const ctx = new Context(message); const command = commandHandler.resolve(ctx); if (command) { try { await middlewarePipeline.execute(ctx); await command.execute(ctx); } catch (error) { await errorHandler.handle(ctx, error); } } });

API Integration Layer

Comprehensive integration with external services and APIs to extend functionality.

Integrated Services

REST APIs GraphQL WebSockets Webhooks
  • Service Abstraction: Unified interface for all external API calls
  • Rate Limiting: Intelligent request throttling per service
  • Response Caching: Redis-backed cache for frequent requests
  • Fallback Mechanisms: Automatic failover to alternative services
  • Data Transformation: Normalization of API responses for consistent handling
// Example API service wrapper class WeatherService { constructor() { this.cache = new LRUCache({ max: 100, ttl: 3600000 }); } async getForecast(location) { const cacheKey = `weather:${location}`; const cached = this.cache.get(cacheKey); if (cached) return cached; const response = await axios.get(API_ENDPOINT, { params: { q: location }, timeout: 5000 }); const forecast = this.transformResponse(response.data); this.cache.set(cacheKey, forecast); return forecast; } }

Security Implementation

Multi-layered security approach to protect user data and prevent abuse.

Security Measures

Encryption Rate Limiting Input Sanitization Audit Logging
  • End-to-End Encryption: All sensitive data encrypted at rest and in transit
  • Permission System: Granular access control for commands and features
  • Anti-Spam: Behavior-based spam detection and prevention
  • Input Validation: Strict validation of all user-provided data
  • Audit Trails: Comprehensive logging of all administrative actions
